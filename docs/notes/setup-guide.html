Setting Up a Quarto Blog with BioPod Theme

A comprehensive guide to creating a Quarto blog from scratch, with or without the BioPod custom theme.

Quick Start

If you just want to use the BioPod theme for your own blog:

  1. Copy these files to your Quarto project:

    • biopod-theme.scss (the custom theme)
    • fonts/ directory (if using custom fonts)

  2. Update your _quarto.yml:

    format:
  html:
    theme: biopod-theme.scss

  3. Render your site:

    quarto render

Setting Up Quarto from Scratch

Prerequisites

  • Quarto installed on your system
  • A text editor (VS Code, RStudio, etc.)
  • Basic understanding of Markdown

Step 1: Create a New Project

# Create project directory
mkdir my-blog
cd my-blog

# Initialize a Quarto website
quarto create project website .

Step 2: Basic Project Structure

A minimal Quarto blog needs:

my-blog/
├── _quarto.yml          # Site configuration
├── index.qmd            # Home page
├── about.qmd            # About page
├── posts/               # Blog posts folder
│   └── post1/
│       └── index.qmd
└── images/              # Images folder

Step 3: Configure _quarto.yml

This is your site’s control center:

project:
  type: website
  output-dir: docs        # For GitHub Pages

website:
  title: "My Blog"
  navbar:
    background: "#e0cdc5"
    left:
      - text: "HOME"
        href: index.qmd
      - text: "BLOG"
        href: posts.qmd
      - text: "ABOUT"
        href: about.qmd

format:
  html:
    theme: default        # Or your custom theme
    toc: true
    toc-location: right

Step 4: Create Your First Post

Create posts/my-first-post/index.qmd:

---
title: "My First Post"
author: "Your Name"
date: "2025-01-28"
categories: [news, tutorial]
description: "A brief description"
---

## Introduction

Your content here...

Step 5: Build & Preview

# Preview locally
quarto preview

# Build for production
quarto render

Using the BioPod Theme

What is BioPod Theme?

BioPod is a warm, academic-styled theme designed for science and research blogs. It features:

  • Soft beige/cream color palette (#e0cdc5, #d5c3b3)
  • Custom fonts (Crimson Text + Apricots display font)
  • Clean, readable code blocks
  • Responsive design
  • Customizable page backgrounds

Installation

  1. Copy theme file:

    # Copy biopod-theme.scss to your project root
cp path/to/biopod-theme.scss ./

  2. Copy fonts (if using custom fonts):

    # Copy the fonts directory
cp -r path/to/fonts ./

  3. Update _quarto.yml:

    format:
  html:
    theme: biopod-theme.scss

Customizing BioPod Theme

The theme is built with SCSS variables that are easy to modify:

// Brand Colors (lines 16-19 in biopod-theme.scss)
$bg-primary: #e0cdc5;      // Main background (header/footer)
$bg-secondary: #d5c3b3;    // Page backgrounds
$text-heading: #323619;    // Headings color
$text-body: #000000;       // Body text color

To customize: 1. Open biopod-theme.scss 2. Find the /*-- scss:defaults --*/ section at the top 3. Modify the color variables 4. Run quarto render to see changes

Creating Your Own Custom Theme

Understanding Quarto Themes

Quarto uses SCSS (Sassy CSS) for theming. A theme file has two main sections:

/*-- scss:defaults --*/
// Variables and settings

/*-- scss:rules --*/
// Custom CSS rules

Step 1: Create Theme File

Create my-theme.scss in your project root:

/*-- scss:defaults --*/

// Import fonts
@import url('https://fonts.googleapis.com/css2?family=Your+Font&display=swap');

// Define colors
$bg-primary: #yourcolor;
$text-body: #yourcolor;
$link-color: #yourcolor;

// Typography
$font-family-sans-serif: 'Your Font', sans-serif;
$font-size-base: 1rem;

/*-- scss:rules --*/

// Custom styling
body {
  font-family: $font-family-sans-serif;
  background-color: $bg-primary;
}

Step 2: Key Variables to Customize

Colors: - $body-bg - Page background - $body-color - Text color - $link-color - Link color - $navbar-bg - Navigation background

Typography: - $font-family-sans-serif - Body font - $headings-font-family - Heading font - $font-size-base - Base font size - $h1-font-size through $h6-font-size - Heading sizes

Code Blocks: - $code-bg - Code background - $code-color - Code text color

Step 3: Adding Custom Fonts

Option 1: Google Fonts

@import url('https://fonts.googleapis.com/css2?family=Crimson+Text&display=swap');

$font-family-sans-serif: 'Crimson Text', serif;

Option 2: Local Fonts

@font-face {
  font-family: 'MyFont';
  src: url('fonts/myfont.woff2') format('woff2');
  font-weight: normal;
  font-style: normal;
}

$font-family-sans-serif: 'MyFont', sans-serif;

Key Components Explained

Page Layouts

Quarto offers several page layouts:

---
title: "My Page"
page-layout: full     # Options: default, full, article
---
  • default - Standard width with margins
  • full - Full width of page
  • article - Optimized for reading (narrower)

Table of Contents (TOC)

Enable navigation sidebar:

---
title: "My Page"
toc: true
toc-location: right    # Options: right, left, body
toc-depth: 3           # How many heading levels to show
---

Important: TOC won’t generate if headings are inside custom div wrappers (:::).

Blog Listings

Create posts.qmd for a blog index:

---
title: "Blog"
listing:
  contents: posts          # Folder to scan
  sort: "date desc"
  type: default           # Options: default, grid, table
  categories: true
  fields: [date, title, description]
  date-format: "DD MMMM YYYY"
---

Custom Page Classes

Add custom CSS classes to pages:

---
title: "About"
---

::: {.custom-class}
Your content here
:::

Then style in your theme:

.custom-class {
  background-color: #yourcolor;
}

Tips & Tricks

1. Testing Changes Quickly

Use live preview while developing:

quarto preview

Changes to .qmd files reload automatically. For theme changes (.scss), you may need to refresh manually.

2. Organizing Images

Keep images in logical folders:

images/          # Site-wide images (logo, hero)
posts/
  post1/
    images/      # Post-specific images

Reference images relative to the file:

![Description](images/photo.jpg)

3. Code Block Styling

Make code readable with proper contrast:

pre {
  background-color: #f5f5f5;
  border-left: 4px solid $text-heading;

  code {
    color: #1a1a1a;      // Dark text
    font-weight: 600;    // Bold for clarity
  }
}

4. Responsive Design

Always test mobile views. Add media queries:

@media (max-width: 768px) {
  .navbar-brand {
    font-size: 1.5rem !important;
  }
}

5. Hide Elements on Specific Pages

Use CSS selectors:

// Hide TOC on about page only
.about-page #quarto-margin-sidebar {
  display: none !important;
}

6. Custom Navigation

Style navbar in _quarto.yml:

navbar:
  background: "#e0cdc5"
  foreground: "#323619"
  left:
    - text: "HOME"
      href: index.qmd

7. Different Backgrounds for Different Pages

Target pages by unique IDs:

// Resources page has unique section ID
body:has(#machine-learning-deep-learning) {
  background-color: $bg-secondary;
}

Common Issues & Solutions

Issue: TOC Not Appearing

Cause: Headings wrapped in div containers

Solution: Move headings outside custom divs:

---
title: "My Page"
toc: true
---

Content introduction here.

## First Section

More content...

Issue: Custom Fonts Not Loading

Cause: Incorrect path or format

Solution: 1. Check font file exists: ls fonts/yourfont.woff2 2. Verify path in SCSS: url('fonts/yourfont.woff2') 3. Use font-display: swap for better loading

Issue: Theme Changes Not Showing

Cause: Browser cache or render cache

Solution:

# Clear Quarto cache
quarto render --no-cache

# Hard refresh browser (Ctrl+Shift+R or Cmd+Shift+R)

Issue: Blog Thumbnails Not Displaying

Cause: Image field not in listing config OR image path incorrect

Solution: 1. Add to posts.qmd: yaml listing: fields: [image, date, title, description] 2. Add to post frontmatter: yaml image: images/thumbnail.jpg

Issue: CSS Rules Not Applying

Cause: Specificity issues

Solution: Use !important sparingly:

.my-class {
  color: red !important;
}

Or increase specificity:

body .my-class h2 {
  color: red;
}

Advanced Customization

Creating Page-Specific Styles

Method 1: Custom CSS Classes

In your .qmd file:

::: {.inspiration-page}
Content here
:::

In theme:

.inspiration-page {
  background-color: $bg-primary;

  h2 {
    font-family: 'Apricots', cursive;
  }
}

Method 2: Body Has Selector

Target pages by unique content:

body:has(#unique-section-id) {
  background-color: $bg-secondary;
}

Custom Hero Sections

Create eye-catching home page layouts:

::: {.hero-container}
::: {.hero-text}
# Your Title
Subtitle text
:::

::: {.hero-image-container}
![](images/hero.png)
:::
:::

Style with flexbox:

.hero-container {
  display: flex;
  gap: 4rem;
  align-items: center;
}

.hero-text {
  flex: 1;
}

.hero-image-container {
  flex: 0 0 55%;
}

Multiple Font Families

Mix fonts for visual hierarchy:

// Body text
$font-family-sans-serif: 'Crimson Text', serif;

// Display/decorative
.navbar-brand,
.hero-title {
  font-family: 'Apricots', cursive !important;
}

Publishing Your Blog

GitHub Pages

  1. Set output directory in _quarto.yml:

    project:
  output-dir: docs

  2. Render site:

    quarto render

  3. Push to GitHub:

    git add .
git commit -m "Update blog"
git push

  4. Enable GitHub Pages:

    • Go to repository Settings
    • Pages section
    • Source: Deploy from branch
    • Branch: main, folder: /docs

Netlify

  1. Create netlify.toml:

    [build]
  command = "quarto render"
  publish = "_site"

  2. Connect repository to Netlify

  3. Deploy automatically on push

BioPod Theme Reference

Color Palette

Primary Background:   #e0cdc5 (warm beige)
Secondary Background: #d5c3b3 (light tan)
Text Heading:         #323619 (dark olive)
Text Body:            #000000 (black)

Font Stack

  • Body: Crimson Text (serif)
  • Display: Apricots (cursive/decorative)
  • Code: Courier New (monospace)

Page Backgrounds

  • Home: Secondary (#d5c3b3)
  • Blog listing: Secondary
  • Blog posts: Primary (#e0cdc5)
  • About: Primary
  • Resources: Primary
  • Inspiration: Primary

Key Design Decisions

  1. Warm, academic aesthetic - Soft colors reduce eye strain
  2. High contrast code blocks - Bold, dark text on light gray
  3. Serif body font - Better for long-form reading
  4. Generous spacing - Improves readability
  5. Minimal borders - Clean, modern look
