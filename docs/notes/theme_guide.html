ğŸ¨ Your Custom Theme: custom-theme.scss

The theme file has two main sections:

1. /*-- scss:defaults --*/ - Variables & Settings

This is where you control everything about your design:

// Colors
$bg-primary: #e0cdc5;          // Home/Blog background
$bg-secondary: #d5c3b3;        // About/Resources background
$text-heading: #323619;        // Heading color
$text-body: #000000;           // Body text color

// Fonts
$font-family-sans-serif: 'Crimson Text', serif;  // Body font
$headings-font-family: 'Crimson Text', serif;     // Headings font

// Spacing (THIS IS THE KEY!)
$spacer: 1rem;                    // Base spacing unit
$paragraph-margin-bottom: 1rem;   // Space after paragraphs
$headings-margin-bottom: 0.5rem;  // Space after headings

// Font sizes
$h1-font-size: 2.5rem;
$h2-font-size: 2rem;
// ... etc

2. /*-- scss:rules --*/ - Custom Styles

This is where you write actual CSS rules that use the variables above.

ğŸ”§ How to Customize

Change Colors

Edit the color variables in the /*-- scss:defaults --*/ section:

$bg-primary: #your-hex-color;
$text-heading: #your-hex-color;

Adjust Spacing

The theme includes spacing control classes:

  • .tight-spacing - Slightly reduced spacing (margin-bottom: 0.5rem)
  • .very-tight-spacing - Very tight spacing (margin-bottom: 0.1rem, line-height: 1)

Use these in your .qmd files:

<h1 class="very-tight-spacing">Title</h1>
<p>Subtitle right below</p>

Change Font Sizes

Adjust heading sizes in the defaults section:

$h1-font-size: 3rem;    // Make h1 bigger
$h2-font-size: 2.5rem;  // Make h2 bigger

Adjust Image Sizes

Find the relevant section in /*-- scss:rules --*/:

.about-entity .about-image img {
  max-width: 300px;  // Change this value
}

.hero-image-container {
  flex: 0 0 45%;      // Change percentage for hero image width
  max-width: 500px;   // Change max width
}

Add Accent Colors

If you want additional accent colors:

// In defaults section:
$accent-primary: #your-color;
$accent-secondary: #another-color;

// Then use them in rules section:
.special-element {
  background-color: $accent-primary;
  border-color: $accent-secondary;
}

ğŸ”¤ Font Setup

Apricots Font (Already Configured)

The custom Apricots font is loaded like this:

@font-face {
  font-family: 'Apricots';
  src: url('fonts/apricots.woff2') format('woff2');
  font-weight: normal;
  font-style: normal;
  font-display: swap;
}

Important: The fonts/ directory is listed as a resource in _quarto.yml so Quarto copies it to the output directory.

To Add Another Custom Font

  1. Get the .woff2 file
  2. Put it in the fonts/ directory
  3. Make sure _quarto.yml has fonts/ listed as a resource (already done)
  4. Add this to the top of custom-theme.scss:
@font-face {
  font-family: 'YourFontName';
  src: url('fonts/your-font-file.woff2') format('woff2');
  font-weight: normal;
  font-style: normal;
  font-display: swap;
}
  1. Use it in your theme:
$font-family-sans-serif: 'YourFontName', sans-serif;

About the docs/ Folder

DO NOT DELETE IT! This is where Quarto builds your website.

  • docs/ = Your compiled, ready-to-deploy website
  • GitHub Pages serves your site FROM this folder
  • Contains: HTML files, CSS, images, all site assets

Should it be in .gitignore? - âŒ NO for GitHub Pages deployment - âœ… YES if using GitHub Actions to build (it rebuilds on every push)

Your setup: Youâ€™re using GitHub Actions, so: 1. The .gitignore excludes docs/ from version control 2. GitHub Actions builds the site on each push 3. The action deploys the built site automatically

If you want to commit docs/ (alternative approach): 1. Remove docs/ from .gitignore 2. Commit the built site: git add docs/ 3. GitHub Pages serves directly from docs/ folder 4. No need for GitHub Actions

Local Preview

cd /home/rish/rish-blogs
quarto preview

Build Site

quarto render

Quarto will automatically: 1. Compile your .scss file into CSS 2. Generate all your pages 3. Output to docs/ directory

Deploy to GitHub Pages

git add .
git commit -m "Updated custom theme"
git push origin main

GitHub Actions will automatically deploy your site.

ğŸ› Troubleshooting

Font Not Loading

  • Check the font file exists at: fonts/apricots.woff2
  • Verify _quarto.yml has fonts/ listed under project: resources:
  • The path in @font-face should be: url('fonts/apricots.woff2')
  • Clear your browser cache (Ctrl+Shift+R)
  • Re-run quarto preview

Colors Not Changing

  • Make sure you edited the $variable in the defaults section
  • Clear your browser cache (Ctrl+Shift+R)
  • Re-run quarto preview

Spacing Issues

  • Check if inline styles are overriding theme styles
  • Use .very-tight-spacing class for tight spacing
  • Adjust $spacer variable for global spacing changes

Theme Not Applying

  • Verify _quarto.yml has: theme: custom-theme.scss
  • Check for SCSS syntax errors
  • Run quarto render to see error messages

ğŸ’¡ Pro Tips

  1. Use Variables: Define colors/spacing as variables at the top, use them throughout
  2. Mobile First: Test on mobile as you build
  3. Spacing Classes: Use .tight-spacing and .very-tight-spacing liberally
  4. Comments: Add comments in your SCSS to remember what things do
  5. Version Control: Commit changes incrementally so you can roll back if needed
