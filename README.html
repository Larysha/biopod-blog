The BioPod

A personal academic blog focused on computational biology, bioinformatics, and the messy reality of research. Built with Quarto and hosted on GitHub Pages.

Live site: https://larysha.github.io/biopod-blog/

About This Blog

The BioPod is where I document my PhD research journey, share tutorials on computational methods, and write about the intersection of biology, data science, and climate adaptation. It’s part learning log, part science communication experiment.

I’m Larysha Rothmann, a PhD candidate in bioinformatics at Stellenbosch University in South Africa. My research focuses on gene regulation in drought-stressed grapevines - using computational pipelines and deep learning to identify genetic variants that might help crops adapt to climate change before testing them experimentally.

This blog serves as: - A record of what I’m learning (mostly bioinformatics tools and methods) - A platform for making computational biology more accessible - Science communication practice for potential career paths beyond academia - Required documentation for my funding applications (academic outreach component)

The BioPod Theme

This site uses a custom Quarto theme designed for readability and academic aesthetics. The theme features:

  • Warm colour palette: Soft beige/cream tones (#e0cdc5, #d5c3b3) instead of stark white backgrounds
  • Custom typography: Crimson Text for body text, Apricots display font for decorative elements
  • Readable code blocks: High contrast with bold text for clear syntax highlighting
  • Flexible layouts: Different background colours for different page types

Using This Theme

You’re welcome to use the BioPod theme as a starting point for your own Quarto site. Here’s what you need:

Required files: - biopod-theme.scss - The main theme file

Optional files: - fonts/apricots.woff2 - Custom display font (used for site title)

How to use it:

  1. Copy biopod-theme.scss to your Quarto project root

  2. (Optional) Copy the fonts/ directory if you want to use the Apricots font

  3. Update your _quarto.yml:

    project:
  type: website
  resources:
    - fonts/  # Only if using custom fonts

format:
  html:
    theme: biopod-theme.scss

  4. Render your site: quarto render

Please: - Credit the original by linking back to this repo or the live site - Make it your own - tweak colours, swap fonts, adjust spacing - Don’t just copy it verbatim (academic sites should reflect the person behind them)

Customizing the Theme

Open biopod-theme.scss and edit the variables in the /*-- scss:defaults --*/ section:

// Change colours
$bg-primary: #e0cdc5;      // Your preferred colour
$bg-secondary: #d5c3b3;    // Your preferred colour
$text-heading: #323619;    // Your preferred colour

// Change fonts
$font-family-sans-serif: 'Crimson Text', serif;  // Your preferred font

See the blog post on setting up this site for detailed instructions.

Development Workflow

This site uses GitHub Actions for automated deployment. The workflow is:

  1. Edit .qmd files locally
  2. Preview changes: quarto preview
  3. Commit and push: git add . && git commit -m "Message" && git push
  4. GitHub Actions automatically renders and deploys the site

No need to run quarto render manually or commit the built site - GitHub handles it.

Tech Stack

  • Framework: Quarto (open-source scientific publishing system)
  • Styling: Custom SCSS theme extending Bootstrap 5
  • Hosting: GitHub Pages (free static hosting)
  • Deployment: GitHub Actions (automated builds on push)
  • Version control: Git + GitHub

Why Quarto? - Native support for R/Python code execution - Built-in citation management (BibTeX) - Outputs to HTML, PDF, presentations, books - Same workflow for lab reports and blog posts - Version-controlled plain text files
